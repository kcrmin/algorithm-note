# Lazy 기법 정리

> **한 줄 요약:** 실제 계산이나 갱신을 즉시 하지 않고 **필요한 순간까지
> 미루는 최적화 기법**

## Lazy 기법이란?

Lazy(지연) 기법은 **연산을 즉시 수행하지 않고, 실제로 값이 필요할 때
처리하는 방식**이다.\
불필요한 계산을 줄여 **시간 복잡도를 줄이거나 성능을 개선하는 것이
목적**이다.

주로 다음 상황에서 사용된다.

-   동일한 계산이 반복될 수 있을 때
-   모든 결과가 실제로 필요하지 않을 때
-   갱신 연산이 매우 빈번할 때

대표적으로 **Lazy Propagation(세그먼트 트리)**에서 많이 사용된다.

------------------------------------------------------------------------

## 대표적인 Lazy 패턴

### 1. Lazy Propagation (Segment Tree)

구간 갱신을 즉시 모든 노드에 반영하지 않고\
**lazy 배열에 미뤄두었다가 필요할 때 전파한다.**

``` java
void propagate(int node, int start, int end) {
    if (lazy[node] != 0) {
        tree[node] += (end - start + 1) * lazy[node];

        if (start != end) {
            lazy[node * 2] += lazy[node];
            lazy[node * 2 + 1] += lazy[node];
        }

        lazy[node] = 0;
    }
}
```

핵심 아이디어

-   **갱신을 기록만 해두고**
-   **조회 또는 하위 접근 시 실제 반영**

------------------------------------------------------------------------

### 2. 필요할 때 계산 (Lazy Evaluation)

값을 바로 계산하지 않고 **처음 접근할 때 계산 후 캐싱**한다.

``` java
Integer value = null;

int getValue() {
    if (value == null) {
        value = expensiveCalculation();
    }
    return value;
}
```

장점

-   불필요한 연산 제거
-   최초 1회만 계산

------------------------------------------------------------------------

### 3. 지연 초기화 (Lazy Initialization)

객체 생성 비용이 클 때 **실제 사용 시점까지 생성 지연**

``` java
class DataHolder {
    private Data data;

    Data getData() {
        if (data == null) {
            data = new Data();
        }
        return data;
    }
}
```

대표 사용 예

-   Singleton 패턴
-   캐시 객체

------------------------------------------------------------------------

## Pruning과 차이

  기법      목적             핵심 아이디어
  --------- ---------------- ----------------------------
  Pruning   탐색 공간 축소   가능성 없는 분기 제거
  Lazy      연산 비용 감소   계산/갱신을 필요할 때 수행

------------------------------------------------------------------------

## 주의사항

-   Lazy 상태가 **정확히 전파되는 시점**을 보장해야 한다
-   동시성 환경에서는 **Race Condition** 주의
-   캐싱 시 **메모리 사용량 증가** 가능

------------------------------------------------------------------------

## 요약

Lazy 기법은\
**"지금 하지 말고, 필요할 때 하자"** 라는 전략으로

-   불필요한 계산을 줄이고
-   전체 연산 비용을 최적화하는 기법이다.
