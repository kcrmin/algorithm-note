# Binary Search Boundary Patterns

## Pattern 1 — First `true`

### Condition Pattern

```
false false false true true
```
### Goal

Find the **first `true`**
### Code

```java
while (lo < hi) {
    int mid = (lo + hi) / 2;
    if (check(mid))
        hi = mid;
    else
        lo = mid + 1;
}
return lo;
```
### Key Idea

- `check(mid) == true` → mid could be the answer → move **left**

- `check(mid) == false` → answer must be **right**

## Pattern 2 — Last `true`

### Condition Pattern

```
true true true false false
```
### Goal

Find the **last `true`**
### Code

```java
while (lo < hi) {
    int mid = (lo + hi + 1) / 2;
    if (check(mid))
        lo = mid;
    else
        hi = mid - 1;
}
return lo;
```
### Key Idea

- `check(mid) == true` → maybe there is a later `true` → move **right**

- `check(mid) == false` → answer must be **left**
### Important

`mid` must be **right-biased**

```
mid = (lo + hi + 1) / 2
```

This prevents **infinite loops** when `lo + 1 == hi`.
